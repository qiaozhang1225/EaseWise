# Stability Taxonomy

## Public Verdicts

稳定性用户侧只允许出现四类结论：

- `适合长期使用`
- `可以继续使用，但要注意使用方式`
- `不建议继续长期主用`
- `不建议长期使用，请尽快调整`

## Verdict Meaning

### 适合长期使用

整体承接力较好，硬伤不重，长期使用不容易反复折腾。
适合作为常用号或重要场景用号继续坚持。

### 可以继续使用，但要注意使用方式

号码不是不能用，但存在明确的扣分点。
适合有条件地继续使用，不适合所有场景都硬当主号。
如果用户特别看重扣分项对应的方向，需要降低结论。

### 不建议继续长期主用

号码仍可能在某些场景短期可用，但不适合继续承担主要号码角色。
尤其当盘面弱项落在事业、财富、婚姻、健康等重要方向时，长期主用容易放大问题。

### 不建议长期使用，请尽快调整

整体承接弱，硬伤集中，长期使用容易持续消耗或反复出问题。
建议尽快调整，不要继续硬扛。

## Element Check Keys

稳定性内部 element_check 使用以下八项：

- `整体承接`
- `宫门关系`
- `四害`
- `特殊组合`
- `神星门`
- `天干/地干`
- `风险敏感方向`
- `最终使用结论`

## Taxonomy Rule

- verdict 由代码锁定或由代码给出候选结论，大模型不能随意改档。
- content 只能解释 locked facts，不能重新算分。
- element_check 是中间判断层，用户主展示只看 verdict 和 content。
