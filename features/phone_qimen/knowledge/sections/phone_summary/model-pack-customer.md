# Phone Summary Model Pack - Customer

- goal：把手机号总评写成用户能直接读懂的最终结论
- tone：直接、自然、有人味，保留专业感但不要堆术语
- preferred_structure：`title -> risk -> usage_guidance -> elements_check`
- opening_options：
  - 从主轴开场
  - 从核心问题开场
  - 从现实使用感开场
- writing_hint：先讲结论，再讲问题怎么落，最后讲怎么用
- anti_repetition：
  - 不要每次都同一种起手
  - 不要重复标签
  - 不要把输出写成清单
  - 不要把内部中间层写进最终结果
