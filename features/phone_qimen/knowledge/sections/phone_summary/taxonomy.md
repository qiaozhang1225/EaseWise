# Phone Summary Taxonomy

## Output shape

1. `title`
2. `risk`
3. `usage_guidance`
4. `elements_check`

## Internal reading order

- `宫`：底盘和大环境
- `门`：现实动作和使用方式
- `神`：气场、助力、远近节奏
- `星`：天时、对象、潜意识、外显气质
- `天干 / 地干`：后段格局、内部牵扯、具体应事
- `特殊组合`：优先级高的直接信号
- `四害`：改写现实表现的硬伤层
- `封顶原因`：限制上限的结构层

## Writing behavior

- `title` 要像一句判断，不像摘要
- `risk` 要讲“问题怎么落”
- `usage_guidance` 要讲“整体怎么用”
- `elements_check` 要保留分层判断，方便后续追踪模型如何看宫、门、神、星、干、特殊组合和四害
- 不要把输出写成中性说明书
- 不要让三个字段互相重复
