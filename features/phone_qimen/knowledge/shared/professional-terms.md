# Professional Terms Readability Protocol

## Core Rule

专业词要保留，但不能裸露成结论。手机号评测的用户可见文案必须遵守：

- 先点名术语，再马上翻译成白话，再落到现实场景。
- 术语不能连续堆叠；一段主文案最多自然带出 1-2 个核心术语。
- 不能只写“某某明显”“某某较重”“某某好/差”，必须说明用户现实里会怎么感受到。
- 如果一个术语解释后仍然很硬，就只保留现实表现，不强行写术语。

推荐句式：

- `这里有“双关”信号，也就是同一组信息同时牵动两个主题，现实里容易一件事没处理好，会连带影响关系或钱。`
- `这里有“空亡”，意思是事情容易落空或拖延，现实里要多确认承诺、时间和回款。`

## Must-Explain Phone Terms

- `双关`：同一组信号同时牵动两个主题，现实里容易一件事连带另一件事。
- `空亡`：事情容易落空、拖延、说了不算，或需要反复确认。
- `门迫`：想推进但被流程、环境或对方条件压住，有力使不出。
- `入墓`：前面能启动，后面收口慢，钱、关系或结果容易压住不动。
- `击刑`：关键处容易别扭、出错、受压，或临场被打断。
- `反吟`：事情容易反复、折返、推倒重来。
- `伏吟`：状态容易停住、重复、拖延，同一问题反复出现。
- `四害`：空亡、门迫、入墓、击刑这类会让现实推进变难的硬伤信号。
- `合十`：表面像配合，实际容易有拉扯或冲突，要看关系、合作和收尾是否稳定。
- `宫`：环境和平台，不要只写宫名，要说这个环境能不能帮用户推进事情。
- `门`：行动方式和通道，要说做事节奏、沟通方式或推进方式。
- `神`：隐性助力和心理状态，要说用户能感受到的气场、贵人或不透明压力。
- `星`：气质、机会和对象侧，要说性格表现、机会类型或对方状态。
- `天干/地干`：后段牵扯和收尾结果，要说兑现、回款、关系后续或内部压力。

## Internal Phrases Not For Public Copy

下面这些是内部分析词，不要出现在 `title`、`content`、`risk`、`usage_guidance` 中：

- 结构承接
- 可用空间
- 可用之处
- 现实承接感
- 底层结构
- 后段承接
- 结构封顶
- 专业层
- 输出合同
- locked facts

可替代表达：

- `结构承接` -> `现实里能不能稳住机会、把事情做成`
- `可用空间` -> `有优势，但要注意怎么使用`
- `后段承接差` -> `开始不难，但收尾、回款或关系维护容易反复`
- `结构封顶` -> `上限被某些风险压住，不能只看表面的亮点`
