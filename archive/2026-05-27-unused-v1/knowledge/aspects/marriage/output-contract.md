# Marriage Output Contract

## Required fields

When this aspect is rendered as structured output, keep these fields:

- `marriage_level`
- `marriage_type`
- `core_judgement`
- `real_world_manifestation`
- `advice`
- `user_facing_paragraph`

## Paragraph rule

最终展示段落要像对用户说的话，不能像：
- taxonomy 标签堆叠
- 四害清单复述
- 空泛的“婚姻好 / 不好”判断

## Minimum quality bar

用户读完以后，至少应该知道：
- 这个号放在婚姻里整体是稳还是不稳
- 它更像什么类型的婚姻结构
- 现实里更容易怎么表现
- 没到满格时，主要差在哪一层
- 如果用户看重婚姻，这个号能不能继续长期使用
- 如果继续用，关系上更该注意什么
- 如果不适合继续用，为什么不建议继续用
