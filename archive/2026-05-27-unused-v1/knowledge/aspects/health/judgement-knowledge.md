# Health Judgement Knowledge

## Role understanding

当解释 `健康` 时，不要把它写成泛泛的“身体好不好”。

你要解释的是：

- 这个号码当前阶段的健康承载和精力消耗怎么样
- 它更像哪一种节律 / 恢复 / 压力结构
- 现实里最容易在哪些场景出现紧绷、睡不好、恢复慢、越用越透
- 如果用户看重睡眠、精力稳定和恢复力，这个号适不适合继续长期使用

## Thickness rule

健康输出也不能只停在一句判断。

默认拆成 4 层：

1. 主判断：健康承载整体落在哪一档
2. 主矛盾：真正决定消耗感的是哪一层
3. 现实表现：这种结构最容易在哪类场景显出来
4. 使用建议：如果用户看重健康承载，这个号能不能继续长期用

## Usage verdict rule

健康不是只做描述，还要给出明确使用结论。

至少要回答清楚下面其中一种：

- 这个号适合长期坚持使用
- 这个号可以继续用，但更吃节律和恢复管理
- 这个号不建议继续长期主用

## Boundary rule

- 健康默认是“当前状态”的结构判断，不主动扩展成未来具体病程预测
- 健康主题只能写成承压、病象、消耗和恢复倾向，不直接下病名
- 不要把 relationship-only 的 `合十格` 带进健康结论
- 不要把所有专题结论粗暴打包成一句“整体都好 / 都不好”
- 不要用神秘化口吻替代结构解释
