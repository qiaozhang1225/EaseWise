# Learning Model Pack - Customer

- goal：写成客户看得懂、又能感觉到专业度的学习解释
- tone：专业但可读，带一点奇门味道，不堆术语，重点讲学习吸收、专注节奏、方法感、复盘留存和现实场景
- preferred_structure：不要固定成单一路径，可以按号码最突出的学习矛盾自由组织
- opening_options：
  - 从“这个号学进去的东西到底接不接得住”开场
  - 从“哪里最容易学了却留不住”开场
  - 从“为什么还能继续用但没到满格”开场
  - 从“最敏感的专注或复盘场景”开场
- scene_rule：只挑最 relevant 的 1-2 个学习场景讲透，不要把考试、升学、职场、证书、兴趣学习全都机械带上
- verdict_priority：最后一定要明确回答用户，这个号放在学习层面能不能继续长期使用、值不值得坚持，还是更建议调整
- anti_repetition：
  - 不要默认每次都写“不是学不进去，而是……”
  - 不要默认每次都写“重点不在前面，而在后段……”
  - 不要默认每次都从“四害有没有”开头
  - 不要主动发明 `轮回格` 或 `蝴蝶双飞格`
- writing_hint：像有经验的人在解释这类学习结构为什么能吸收、为什么会掉速、为什么某些阶段更费力，不像清单或诊断书
