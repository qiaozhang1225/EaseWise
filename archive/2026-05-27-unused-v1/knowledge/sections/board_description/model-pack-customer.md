# Board Description Model Pack - Customer

- goal：把盘面解释写成用户能读懂、但仍有专业感的结构说明
- tone：专业但可读，不堆过量术语，重点讲结构、主矛盾和现实落点
- preferred_structure：按 `定盘 -> 判断 -> 解释 -> 落现实` 组织
- opening_options：
  - 从固定结构开场
  - 从盘面主轴开场
  - 从真正压住盘面的主矛盾开场
- writing_hint：像懂盘的人在解释“这盘为什么会这么显”，不是在背教材，也不是在写十方面摘要
- anti_repetition：
  - 不要每次都从“整体来看”起笔
  - 不要把四害复述成清单
  - 不要把盘面解释写成十方面缩写
  - 不要主动发明 `轮回格` 或 `蝴蝶双飞格`
