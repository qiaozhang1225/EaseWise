# Board Description Judgement Knowledge

## Role understanding

当解释 `奇门盘解读 / 盘面解释` 时，不是在重复十方面结论，也不是在给用户做一句“好或不好”的总评。

这一步要回答的是：

- 这张盘固定落了什么结构
- 主轴到底偏向哪一侧
- 真正压住盘面落地感的主矛盾在哪里
- 四害、特殊组合、封顶原因会怎么改写现实表现
- 最后这套结构在现实里会更像什么状态

## Thickness rule

盘面解释默认拆成 4 层：

1. `盘面基础结构`
   - 先把后七位与引干 / 宫 / 神 / 星 / 门 / 天干 / 地干交代清楚
2. `核心关系判断`
   - 先看宫门关系、后两干关系，再看四害、特殊组合、结构封顶
3. `专业盘面描述`
   - 只保留一个主轴和一个主矛盾，不要平均摊开
4. `现实落点`
   - 把盘面结构翻译成现实里的表现方式，而不是换个说法再复读术语

## Main contradiction rule

盘面解释最怕“所有东西都提到一点，但没有主线”。

一定要让一个主矛盾贯穿全文，例如：

- 亮点结构还在，但四害或特殊组合把落地层压住
- 盘面能起势，但后段发沉、高压碰撞更明显
- 表面能量不低，但关键层位发空或受压，导致现实承接打折
- 外盘能推，内盘带克，所以越到现实越吃环境与角色匹配

## Evidence boundary

盘面解释只能基于代码层已经锁定的事实展开。

当前可直接使用的证据主要是：

- `board_basis`
- `score_facts`
- `palace_door_relation`
- `stem_pair_relation`
- `four_harms`
- `pattern_flags`
- `risk_pairs`
- `edge_flags`
- `structural_cap_reasons`
- `technical_focus`

所以：

- 可以解释这些事实怎样互相加权
- 不要主动发明 evidence 里没有的格局、门派说法或课程术语

特别是：

- 不要主动写出 `轮回格`
- 不要主动写出 `蝴蝶双飞格`

除非它们真的已经进入 locked facts。

## Distinguish board description from ten aspects

盘面解释不是十方面的总和。

不要把这一段写成：

- 性格一句
- 财运一句
- 事业一句
- 健康一句

那样会把盘面的“结构判断”重新写回“专题摘要”。

盘面解释更像：

- 先说结构怎么立起来
- 再说哪里压住了它
- 最后说现实里为什么会这样显

## Minimum quality bar

用户读完以后，至少应该知道：

- 这张盘的固定结构是什么
- 盘面主轴到底偏向哪里
- 真正决定分高分低、落地顺不顺的主矛盾是什么
- 四害、特殊组合和封顶原因有没有压住亮点
- 现实里这张盘更像哪类状态，而不只是“好 / 坏”两个字
