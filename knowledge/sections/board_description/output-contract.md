# Board Description Output Contract

## Required fields

When board description is rendered as structured output, keep these fields:

- `board_basis`
- `core_relation_judgement`
- `technical_narrative`
- `practical_manifestation`

## Contract rule

这 4 个字段分别对应：

- `盘面基础结构`
- `核心关系判断`
- `专业盘面描述`
- `现实落点`

不要把它们重新混成一段，也不要让其中任何一段空掉。

## Minimum quality bar

用户读完以后，至少应该知道：

- 盘到底怎么落
- 核心关系顺不顺、冲不冲
- 真正的主矛盾在哪一层
- 现实里最像什么表现

## Anti-pattern reminder

不要把输出写成：

- 四害 checklist
- 模板拼接
- 十方面缩写版
- 空泛玄学套话
