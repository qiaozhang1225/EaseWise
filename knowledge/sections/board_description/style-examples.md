# Board Description Style Examples

These examples are style references, not fixed templates.

## Example 1｜先定结构

这张盘先不用急着下十方面结论，第一步是把结构站稳。后七位怎么落，引干、宫、神、星、门、天干、地干怎么排，决定了这盘到底是顺接、带克，还是表面有势头、后面却容易掉速。盘面一旦定错，后面所有解释都会漂。

## Example 2｜先亮点再压制

有些盘表面看起来很亮，因为门、星、神里有能推得动的东西；但真正决定现实落地的，往往不是“有没有亮点”，而是这些亮点有没有被四害、特殊组合或封顶原因压住。盘解如果只讲前半段，就会把真实代价说薄。

## Example 3｜主矛盾写法

盘面解释不要平均用力。真正需要说透的，是哪一层在主导这张盘：是外盘能推、内盘带克，还是前面能起、后段发沉，或者风险感知很强，但一进现实就容易被压力和边界回拉。主矛盾一旦抓准，盘面就会立住。

## Example 4｜现实落点写法

现实落点不是把术语换成白话再说一遍，而是告诉用户：这张盘放到现实里，更像什么状态。是更容易先看到问题、先感到压力，还是更适合慢慢沉淀、把结构做扎实；是外放拉动强，还是边界一乱就开始打折。用户真正想知道的是“这盘在现实里怎么显”。

## Anti-pattern reminder

尤其不要每次都长成同一种写法：

- “总体不错，但也有一点问题”
- “这个盘有好有坏，要综合看”
- “空亡有、门迫无、入墓无、击刑有”
- “事业上如何、财运上如何、感情上如何”

这些都太平，不够像盘解。
