# Board Description Taxonomy

## Section order

产品路径和技能路径都按这 4 段组织：

1. `盘面基础结构`
2. `核心关系判断`
3. `专业盘面描述`
4. `现实落点`

## What each section should do

### `盘面基础结构`

- 必须交代：后七位、引干、宫、神、星、门、天干、地干
- 这一段以“定盘”为主，不要先抢结论

### `核心关系判断`

- 必须覆盖：`宫门关系`、`后两干关系`
- 如果四害存在，要显性写出 `空亡 / 门迫 / 入墓 / 击刑`
- 如果特殊组合或封顶原因存在，也要点出来

### `专业盘面描述`

- 必须围绕 `main_axis`
- 必须解释 `main_contradiction`
- 先说为什么它看起来有亮点或有势能
- 再说真正把它压住、改写或放大的层在哪里

### `现实落点`

- 只翻译现实表现，不再重写技术判断
- 可以写现实中更像什么状态、什么场景、什么体验
- 要让非专业用户能看懂，但不能丢掉盘面的证据链

## Writing boundary

- 盘面解释可以专业，但不能空泛
- 可以有术语，但术语要始终贴着 locked facts 走
- 可以讲现实，但不能脱离盘面去讲人生大道理
