# Learning Output Contract

## Required fields

When this aspect is rendered as structured output, keep these fields:

- `learning_level`
- `learning_type`
- `core_judgement`
- `real_world_manifestation`
- `advice`
- `user_facing_paragraph`

## Paragraph rule

最终展示段落要像对用户说的话，不能像：

- taxonomy 标签堆叠
- 四害清单复述
- 空泛的“学习好 / 不好”判断
- 智商、学历、考试结果的武断断言

## Minimum quality bar

用户读完以后，至少应该知道：

- 这个号码放在学习里整体是顺还是不顺
- 它更像哪一种学习结构
- 现实里最容易怎么表现出来
- 没到满格时，主要差在哪一层
- 如果用户看重学习效率、专注稳定和长期沉淀，这个号能不能继续长期使用
- 如果继续用，最该避开的学习卡点是什么
- 如果不建议继续用，真正的主矛盾落在哪里
