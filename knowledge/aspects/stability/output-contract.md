# Stability Output Contract

## Required fields

When this aspect is rendered as structured output, keep these fields:

- `stability_level`
- `stability_type`
- `core_judgement`
- `real_world_manifestation`
- `advice`
- `user_facing_paragraph`

## Paragraph rule

最终展示段落要像对用户说的话，不能像：
- taxonomy 标签堆叠
- 四害清单复述
- 空泛的“稳/不稳”判断

## Minimum quality bar

用户读完以后，至少应该知道：
- 这个号整体稳不稳
- 属于什么类型的稳或不稳
- 现实里更容易怎么表现
- 没到满格时，主要差在哪一层
- 这个号能不能继续长期使用
- 如果继续用，应该怎么用更合适
- 如果不适合继续用，为什么不建议继续用
